# Limitations / Ограничения

## RU

1. Проект является лабораторной моделью, а не production CI/CD системой.
2. Demo application содержит intentionally vulnerable code.
3. Результаты инструментов требуют human validation.
4. Security gates требуют настройки под риск-профиль организации.
5. False positives и false negatives возможны.
6. SBOM сам по себе не устраняет риски, а даёт inventory для анализа.
7. Qubes OS используется как изолированная рабочая среда, а не как обязательная production-платформа.

## EN

1. The project is a lab model, not a production CI/CD system.
2. The demo application contains intentionally vulnerable code.
3. Tool results require human validation.
4. Security gates must be tuned to an organization's risk profile.
5. False positives and false negatives are possible.
6. SBOM does not eliminate risks by itself; it provides inventory for analysis.
7. Qubes OS is used as an isolated working environment, not as a mandatory production platform.
