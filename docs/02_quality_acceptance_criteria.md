# Quality Acceptance Criteria / Критерии приёмки качества

## RU

Проект считается успешным, если:

- создана воспроизводимая структура репозитория;
- vulnerable-version содержит учебные defects;
- security tools обнаруживают defects;
- policy gate падает на vulnerable-version;
- fixed-version устраняет defects;
- policy gate проходит после remediation;
- есть before/after evidence;
- подготовлены отчёт и презентация.

## EN

The project is successful if:

- the repository structure is reproducible;
- vulnerable-version contains educational defects;
- security tools detect the defects;
- the policy gate fails on vulnerable-version;
- fixed-version remediates the defects;
- the policy gate passes after remediation;
- before/after evidence exists;
- report and presentation are prepared.
